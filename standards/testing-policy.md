# Testing Policy

## Pflichttests

- Unit Tests fuer Business-Logik, Validierungen und Prompt Assembly.
- Integration Tests fuer Agent-Orchestrierung und externe Integrationen.
- API Tests fuer Schnittstellen, Status Codes und Fehlerfaelle.
- Agent Evaluation Tests fuer Rollenverhalten, Grounding und Konsistenz.

## Ziel

Mindestens 80 Prozent Abdeckung fuer kritische Logik. Fuer Agenten muessen qualitative Eval-Kriterien dokumentiert werden.
